# Off-Platform Prevention System - Quick Reference

## For Developers

### Using MessageFilterService

```dart
import 'package:mobilis_by_psdc_app/services/message_filter_service.dart';

// Analyze a message
final analysis = MessageFilterService.analyzeMessage(
  'Hey, my WhatsApp is +1-555-123-4567'
);

// Returns:
// {
//   'should_flag': true,
//   'risk_score': 0.65,
//   'risk_level': 'high',
//   'found_keywords': ['whatsapp', 'phone_number']
// }

// If suspicious, flag it
if (analysis['should_flag'] == true) {
  await MessageFilterService.flagMessageForReview(
    messageId: message.id,
    conversationId: conversation.id,
    senderId: currentUser.id,
    flagReason: 'Found: ${analysis['found_keywords'].join(", ")}',
    messageContent: messageContent,
  );
}

// Check if user is blocked
final flagCount = await MessageFilterService.getUserFlagCount(userId);
if (flagCount >= 3) {
  // User is blocked, prevent sending
}
```

## For Admins

### Finding Flagged Messages

1. **Admin Dashboard** → **Message Review** tab
2. Click **Pending** to see messages awaiting action
3. Review each flag:
   - Sender name and ID
   - Message content
   - Detected keywords
   - Risk level color
4. Choose action:
   - ✓ **Confirm** - Message was violation, user penalized
   - ✕ **Dismiss** - False positive, no action

### User Blocking

- **3 Confirmed Flags** = Automatic block
- **Manual Block** = Admin can block immediately via Dismiss button with "Block User" option
- **Checking Block Status** = See `users.is_blocked` field in database

### Reviewing User History

Currently shows:
- Current flag count
- Recent flagged messages
- Risk levels over time

Future: Full user profile with all violations

## For End Users

### Renter/Partner/Driver

**When You See Warning ⚠️**
```
Warning: This message contains patterns suggesting arranging 
transactions outside the app. Messages are reviewed by admins.
```

- **Don't:** Include phone numbers, emails, or payment methods
- **Do:** Use the app for all booking communication
- **Why:** Protects the Mobilis platform and your commission

**What Gets Flagged:**
- Phone numbers: +1-555-123-4567
- Email addresses: user@example.com
- Messaging apps: WhatsApp, Telegram, Signal
- Meeting locations: "meet at my place"
- Payment methods: Bank account, wire transfer

**What's Safe:**
- "When can I pick up?"
- "What time on Friday?"
- "Is the fuel included?"
- "Can I extend the rental?"

**If Your Account Gets Blocked:**
- After 3 confirmed violations, account is restricted
- Contact support@mobilis.com to appeal
- Show proof your messages were legitimate

## For Database Admins

### Checking Message Flags

```sql
-- See all pending flags
SELECT * FROM message_flags 
WHERE status = 'pending_review'
ORDER BY created_at DESC;

-- See confirmed violations by user
SELECT sender_id, COUNT(*) as violations
FROM message_flags
WHERE status = 'confirmed'
GROUP BY sender_id
HAVING COUNT(*) >= 3;

-- See false positives (dismissed)
SELECT * FROM message_flags
WHERE status = 'dismissed'
LIMIT 20;

-- Check if user is blocked
SELECT id, email, is_blocked, off_platform_flag_count
FROM users
WHERE is_blocked = true;
```

### Tuning Risk Detection

**Current Thresholds:**
- **Low Risk:** < 0.25 (pass through)
- **Medium Risk:** 0.25 - 0.49 (show warning)
- **High Risk:** ≥ 0.50 (flag automatically)

**To Make Stricter:**
Edit `message_filter_service.dart`:
```dart
if (riskScore >= 0.35)  // Stricter than 0.50
```

**To Add Keywords:**
Edit `message_filter_service.dart`:
```dart
static const offPlatformKeywords = [
  // ... existing keywords
  'crypto',  // Add this
  'bank transfer',
];
```

### RLS Policies

```sql
-- Users see their own flags
CREATE POLICY message_flags_select_own
  ON public.message_flags FOR SELECT
  USING (auth.uid() = sender_id);

-- Admins review all flags

```

## Database Schema Quick Ref

```sql
-- Main table for flagged messages
message_flags:
  id (UUID)
  message_id (UUID) → messages.id
  conversation_id (UUID) → conversations.id
  sender_id (UUID) → users.id
  flag_reason (TEXT) - "Detected: whatsapp, phone_number"
  message_content (TEXT) - Original message
  risk_score (0.0-1.0) - Calculated score
  risk_level (low/medium/high)
  status (pending_review/confirmed/dismissed)
  admin_notes (TEXT)
  created_at (TIMESTAMP)
  reviewed_at (TIMESTAMP)

-- User flag tracking
users:
  off_platform_flag_count (INT) - How many times flagged
  is_blocked (BOOLEAN) - Restricted if true

-- Auto-generated message tracking
messages:
  is_auto_generated (BOOLEAN) - System message badge

-- Conversation creation tracking
bookings:
  conversation_created (BOOLEAN) - Prevents duplicates
```

## Common Issues & Solutions

| Issue | Cause | Solution |
|-------|-------|----------|
| Message not flagging | Risk < 0.50 | Adjust thresholds in MessageFilterService |
| Auto-message duplicate | `conversation_created` not checked | Verify BookingService sets flag after creating |
| Warning not showing | ChatDetailScreen not using MessageFilterService | Check ChatDetailScreen implementation |
| Admin can't see tab | Role not 'admin' | Verify `AuthService().getUserRole()` returns 'admin' |
| User not blocked after 3 flags | Flag count not incremented | Check flagMessageForReview() increments counter |
