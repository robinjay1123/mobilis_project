# Off-Platform Transaction Prevention System

## Overview

A comprehensive safety system to prevent users from conducting transactions outside the Mobilis app, similar to Airbnb and Grab's anti-fraud measures.

## Components

### 1. **Message Filter Service** 
**File:** `lib/services/message_filter_service.dart`

Analyzes messages in real-time for off-platform transaction attempts:

**Detection Keywords:**
- Direct contact: WhatsApp, Telegram, phone, call me, text me, DM
- Bank details: Bank account, wire transfer, SWIFT, IBAN
- Payment avoidance: Pay outside, skip payment, avoid commission
- Meeting outside: Meet at, pickup from, my place
- Personal info: Email addresses, phone numbers (regex patterns)

**Risk Scoring:**
- Low: < 0.25 (informational warning)
- Medium: 0.25-0.49 (flagged with warning)
- High: ≥ 0.50 (flagged + user warning)

**Methods:**
```dart
analyzeMessage(messageContent)      // Returns risk analysis
flagMessageForReview(...)           // Creates admin review task
getUserFlagHistory(userId)          // Gets user's flag record
getUserFlagCount(userId)            // Gets number of flags
reviewFlaggedMessage(...)           // Admin approves/dismisses flag
```

### 2. **Auto-Message Service**
**File:** `lib/services/auto_message_service.dart`

Automatically creates conversations and initial messages when bookings are confirmed.

**Features:**
- Creates conversation when booking transitions to "confirmed"
- Sends system message with:
  - Booking details (vehicle, dates, pricing)
  - Safety reminders about app payments
  - Commission warning for off-platform attempts
  - Link to support
- Marks messages as `is_auto_generated` for UI distinction

**Methods:**
```dart
createBookingConversation(...)      // Creates convo + auto-message
getAutoMessages(conversationId)     // Lists system messages
isConversationAutoGenerated(...)    // Checks if auto-created
```

### 3. **Chat Detail Screen**
**File:** `lib/mobile_ui/screens/home/chat_detail_screen.dart`

Enhanced chat UI with safety features:

**Features:**
- Message list with sender differentiation
- Auto-message badge 🤖 for system messages
- Real-time message analysis on send
- Warning banner if suspicious content detected
- Flag notification when message is flagged
- Blocks sending if user is blocked

**User Experience:**
1. User types message
2. System analyzes content in real-time
3. If suspicious: Shows warning ⚠️
4. If high-risk: Message flagged + notification sent to admins
5. User can still send (not blocked at send)
6. Admin reviews flag and can restrict user if necessary

### 4. **Admin Message Review Screen**
**File:** `lib/mobile_ui/screens/admin/message_review_screen.dart`

Admin dashboard for reviewing flagged messages:

**Tabs:**
- **Pending:** Messages awaiting review
- **Confirmed:** Confirmed off-platform attempts (action taken)
- **Dismissed:** False positives

**Admin Actions:**
- **Confirm:** User may be warned/restricted (3 strikes = block)
- **Dismiss:** False positive, continue monitoring
- **Block User:** Immediately restrict from app

### 5. **Database Changes**
**File:** `supabase/migrations/20260419120000_create_message_flags.sql`

**New Table: `message_flags`**
```sql
id (UUID) - Flag ID
message_id (UUID) - Message being flagged
conversation_id (UUID) - Conversation context
sender_id (UUID) - User who sent message
flag_reason (TEXT) - Why flagged
message_content (TEXT) - Message text
risk_score (0-1) - Risk calculation
risk_level (low/medium/high) - Category
status (pending_review/confirmed/dismissed) - Review status
admin_notes (TEXT) - Admin feedback
created_at - When flagged
reviewed_at - When reviewed
```

**Users Table Updates:**
- `off_platform_flag_count` - Number of times flagged
- `is_blocked` - User can't use app if true

**Messages Table Updates:**
- `is_auto_generated` - Is this a system message?

**Bookings Table Updates:**
- `conversation_created` - Has convo been auto-created?

## Integration Flow

### When User Books Vehicle:

```
1. Booking created
   ↓
2. Booking confirmed by partner/driver
   ↓
3. AutoMessageService.createBookingConversation() called
   ↓
4. Conversation created with participants
   ↓
5. System message sent with booking details + safety warning
   ↓
6. Both parties see auto-generated badge 🤖 on first message
   ↓
7. Users can now message freely
```

### When User Sends Message:

```
1. User types message
   ↓
2. MessageFilterService.analyzeMessage() checks content
   ↓
3. If suspicious (risk_score >= 0.5):
   a. Show warning banner to user
   b. Send anyway (don't block)
   c. Flag for admin review
   d. Notify user: "Message flagged for safety"
   ↓
4. If not suspicious:
   a. Send normally
   b. No warning shown
```

### When Admin Reviews Flag:

```
1. Admin sees flagged message in AdminMessageReviewScreen
2. Reviews:
   - Sender name & history
   - Message content
   - Risk keywords detected
   - Risk score & level
3. Takes action:
   a. CONFIRM: Message was violation
      - User flag_count ++
      - If flag_count >= 3: User blocked
   b. DISMISS: False positive
      - Continue monitoring
   c. BLOCK: Immediate action
      - User set is_blocked = true
```

## User Experience

### For Renters:
- Book vehicle
- Conversation auto-opens
- See system message with booking details
- Can message partner/driver
- If they try off-platform contact: Warning shown, message flagged
- Multiple violations = account restricted

### For Drivers/Partners:
- See booking notification
- Conversation auto-created
- See renter in chat
- Can negotiate pickup/dropoff details
- If they try off-platform contact: Same warning system
- Multiple violations = account restricted

### For Admins:
- Review Hub tab
- See all flagged messages
- Review context and keywords
- Confirm violations or dismiss false positives
- Take action (warn, restrict, block)

## Configuration

### To Adjust Risk Thresholds:
Edit in `message_filter_service.dart`:
```dart
// Change risk levels
riskScore += 0.15;  // Keyword weight
riskScore += 0.25;  // Phone number weight
riskScore += 0.20;  // Email weight

// Change flag trigger
if (riskScore >= 0.5)  // Change to 0.3 for stricter
```

### To Add Keywords:
Edit in `message_filter_service.dart`:
```dart
static const offPlatformKeywords = [
  // Add new keywords here
  'your_keyword_here',
];
```
